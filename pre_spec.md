## 機能

AI-Aging-Generatorで「老化を受け入れるアプリ」

## API

https://docs.perfectcorp.com/develop/introduction

## 仕様・狙い

- Aging Generator + LLM
- 美容・ファッションに限らず広い視点で活用したアイデア
- 顔の消去など追加処理が不要なのでAPIを正面から使い切れる。
- アウトプットのクオリティ：高。
- 10年後・20年後・30年後の自分の画像が並ぶビジュアルは単純に強い。そこにLLMが生成した「老いた自分からの返答」テキストが乗ると、プロダクトとして完成度が高く見える。 
- 単純な「老化フィルター作りました」では埋もれる
- Aging Generatorが返す「老いた自分の画像」を複数並べ、LLMが各年齢に対応した手紙文を生成する。年齢の段階指定ができなければ「APIが返してくれた段階をそのまま使う」と記事の中で正直に書いても成立する。むしろ「APIの仕様に引っ張られて作った」という正直さは記事の誠実さになる。
- Future Self Interviewの強さ
- Elder Letterより強い理由がある。インタビュー形式にすると「ユーザーが質問を入力する」というインタラクションが生まれ、デモとして触ったときの体験が段違いに上がる。審査員がデモを触ったときの印象は点数に直結する。

## 仕様案

1. 自撮りアップロード
2. Aging GeneratorでAPIが返す複数の段階の老いた自分を生成・表示
3. ユーザーが未来の自分に質問を入力（「転職して後悔してる？」「もっとやっておけばよかったことは？」）
4. LLMが老いた自分の人格で回答を生成（プロンプトに年齢・顔から読み取れる雰囲気を渡す）
5. 会話として続けられる
